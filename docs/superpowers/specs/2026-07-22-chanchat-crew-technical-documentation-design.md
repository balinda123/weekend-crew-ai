# 馋猫局儿技术文档设计记录

## 已确认的范围

- MVP 面向北京及周边朋友的美食短途组局，支持 4–10 人。
- 首版包含地点收集、地图、AI 行程、AA、私密攻略与复刻。
- AI 从 v0.1 开始参与截图结构化和行程编排，但不能代替确定性路线与账本逻辑。
- 产品角色为白色田园猫搭档：芝士负责路线/下一站，奶糖负责账本/攻略状态。
- 地图服务默认使用高德；通过 `MapProvider` 隔离，未来可替换。

## 文档结构决策

采用三份文档而不是只拆成“前端一份、后端一份”：

1. `TECHNICAL-DESIGN.md` 是共同真相，定义架构、实体、数据流、安全边界、成本与 API 原则。
2. `FRONTEND-IMPLEMENTATION.md` 定义小程序页面、组件、状态与测试责任。
3. `BACKEND-IMPLEMENTATION.md` 定义 API、数据、提供商适配、AI 管道与测试责任。

这种结构避免前后端各自维护一份接口定义，导致字段、权限、异步状态不一致。
