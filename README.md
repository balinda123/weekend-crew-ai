# Weekend Crew AI（周末局）

面向朋友周末短途出行的 AI 协作工具：把刷到的地点、共同决策、地图行程、AA 结算和可复刻攻略放进同一个组局。

## 为什么做它

一群朋友规划周末出游时，常常需要在内容平台、地图协作工具和 AA 记账工具之间来回切换；多人协作又经常受人数限制。Weekend Crew AI 的目标是让 4–10 人免费完成一次完整闭环：

```text
收藏地点 → 共同决策 → AI 排行程 → 现场记账 → AA 结算 → 分享/复刻攻略
```

## 核心功能

- 多人组局、地图收藏、地点状态与投票
- 截图 OCR + 地图 POI 候选匹配，需用户确认后入库
- AI 多约束行程规划：理解人数、日期、预算、出发地与偏好；生成可编辑方案
- AA 账本：支持多人参与、均分/按份额分摊和债务化简结算
- AI 攻略：将一次真实组局生成私密分享攻略；他人可按自己的条件一键复刻为新组局

## AI 设计原则

- AI 是核心能力，不是附加聊天框。
- OCR、POI 匹配、路线约束计算和 LLM 工具调用组成可验证流程。
- 行程排序先由确定性距离、时间和营业约束生成候选，LLM 负责理解、编排和解释。
- AI 输出必须通过 JSON Schema 校验；未确认地点不会进入正式行程。
- 使用缓存、限流、低置信度回退和成本监控控制模型调用成本。

## 技术规划

| 层 | 方案 |
|---|---|
| 前端 | Taro + React + TypeScript + NutUI React Taro |
| 后端 | Python + FastAPI |
| 数据 | PostgreSQL + PostGIS |
| AI | OCR、地图 POI 匹配、LLM 工具调用、结构化输出校验 |
| 接口契约 | OpenAPI，生成前端 TypeScript 类型 |

项目采用 Monorepo：前后端独立开发和部署，但共享接口契约、文档与工程规范。
